# Портреты наследников / султанов по поколениям

Цепочка престолонаследия: стартовый султан (поколение 0) → его сын (поколение 1) →
сын сына (поколение 2) → … Каждое поколение, начиная с 1-го, — уникальный человек
со своими фото (их присылает заказчик).

Конвенция имён (аспект 3:4, .png):
- `heir_<gen>_teen.png`  — ПОДРОСТКОВОЕ фото. Показывается в семейном древе,
  пока этот сын — наследник при отце (поколении gen-1).
- `heir_<gen>_adult.png` — ВЗРОСЛОЕ фото. Главный портрет, когда сын взошёл на трон
  (стал правителем поколения gen). Должно быть тем же лицом, что и teen, но старше.

Где `<gen>` — номер поколения (1, 2, 3, …).

Примеры:
- Первый сын стартового султана: `heir_1_teen.png` + `heir_1_adult.png`.
- Его сын: `heir_2_teen.png` + `heir_2_adult.png`.

Поколение 0 (стартовый султан) использует существующий `assets/art/sultan.png`
и `sultan_portrait.png` — отдельные файлы для него не нужны.

Пока фото поколения нет — в древе и на экране Династии рисуется фолбэк-медальон
с инициалом. Godot создаёт .import для новых .png при первом открытии проекта.
